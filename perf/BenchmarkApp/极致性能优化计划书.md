### 1.3 Environment.ProcessorCount 配置项分析

当前使用 `Environment.ProcessorCount` 的配置：

| 配置项 | 默认值 | 位置 | 影响范围 | 性能影响 |
|--------|--------|------|----------|----------|
| DispatcherThreadCount | ProcessorCount | DispatcherOptions:528 | 消息调度线程池 | 🔴 关键 |
| ResponseProcessorThreadCount | ProcessorCount / 2 | ResponseProcessorOptions:588 | 响应处理线程池 | 🔴 关键 |
| WorkerThreads (生产环境) | ProcessorCount * 2 | ServerConfiguration:149 | 工作线程池 | 🟡 重要 |
| ParallelismDegree | ProcessorCount | MetricsAnalyzer:845 | 指标分析并行度 | 🟢 次要 |
| MaxConcurrentOperations | ProcessorCount * 2 | MetricsConfiguration:164 | 指标并发上限 | 🟢 次要 |

**核心问题识别**:

1. **线程过度配置** (ProcessorCount * 2)
   - 在高并发场景下导致线程上下文切换开销增大
   - 线程竞争导致 CPU 缓存失效率上升
   - 建议: 根据 I/O 密集型 vs CPU 密集型工作负载动态调整

2. **响应处理线程不足** (ProcessorCount / 2)
   - 在高吞吐场景下成为瓶颈
   - 响应队列积压导致延迟上升
   - 建议: 至少使用 ProcessorCount，并支持动态扩展

3. **未考虑 NUMA 架构**
   - 现代服务器多采用 NUMA 架构
   - ProcessorCount 不区分物理核心 vs 逻辑核心
   - 建议: 使用 NUMA 感知的线程分配策略

```csharp
public class BatchedTransport : ITransport
{
    private readonly Channel<ReadOnlyMemory<byte>> _sendQueue;
    private const int MaxBatchSize = 64;

    private async Task BatchSendLoop()
    {
        var batch = new ReadOnlyMemory<byte>[MaxBatchSize];

        while (await _sendQueue.Reader.WaitToReadAsync())
        {

        }
    }
}
```

##### 方案 1: 分层内存池 (已部分实现)

**改进点**:
```csharp
public class TieredMemoryPool
{
    // L1: 线程本地缓存 (无锁)
    [ThreadStatic]
    private static ThreadLocalCache? _l1Cache;

    // L2: NUMA 本地缓存 (减少跨 NUMA 访问)
    private readonly NumaLocalCache[] _l2Caches;

    // L3: 全局共享池 (最后的回退)
    private readonly GlobalBufferPool _l3Pool;

    public byte[] Rent(int size)
    {
        // 1. L1 缓存 (最快，无锁)
        if (_l1Cache?.TryRent(size, out var buffer) == true)
            return buffer;

        // 2. L2 NUMA 本地缓存 (次快，减少跨 NUMA 开销)
        var numaNode = Thread.GetCurrentProcessorId() / CoresPerNumaNode;
        if (_l2Caches[numaNode].TryRent(size, out buffer))
            return buffer;

        // 3. L3 全局池 (最慢，但保证可用)
        return _l3Pool.Rent(size);
    }
}
```

```csharp
// 为消息对象提供对象池
public class ObjectPool<T> where T : class, new()
{
    private readonly ConcurrentBag<T> _pool = new();
    private readonly Func<T> _factory;
    private readonly Action<T>? _reset;

    public T Rent()
    {
        if (_pool.TryTake(out var item))
        {
            _reset?.Invoke(item);
            return item;
        }
        return _factory();
    }

    public void Return(T item)
    {
        if (_pool.Count < MaxPoolSize)
            _pool.Add(item);
    }
}

    reset: e => e.Reset()
```

```csharp
    [FieldOffset(28)] public MessageStatus Status;    // 4 bytes
    [FieldOffset(32)] public int ConnectionIdHash;    // 4 bytes (代替 string)
    [FieldOffset(36)] private int _padding;           // 填充到 64 字节
    // Payload 和 Header 通过指针引用，避免缓存行污染
}

// 避免伪共享 (False Sharing)
[StructLayout(LayoutKind.Explicit, Size = 128)]
public struct PaddedCounter
{
    [FieldOffset(0)] private long _value;
    // 填充到 128 字节 (2 个缓存行) 避免与其他计数器共享缓存行
}
```

##### 方案 2: 内存预取 (Prefetching)

```csharp
// 在处理消息前预取下一个消息的数据
public async Task ProcessBatch(ReadOnlyMemory<MessageSlot> batch)
{
    for (int i = 0; i < batch.Length; i++)
    {
        ref var current = ref batch.Span[i];

        // 预取下一个消息的数据到缓存
        if (i + 1 < batch.Length)
        {
            ref var next = ref batch.Span[i + 1];
            System.Runtime.Intrinsics.X86.Sse.Prefetch0(ref next);
        }

        await ProcessMessage(current);
    }
}
```

- 缓存命中率提升 **15-25%**
- 延迟降低 **10-15%**

```csharp
// 使用 SIMD 加速内存拷贝和比较
public static class SimdHelpers
{
    [MethodImpl(MethodImplOptions.AggressiveInlining)]
    public static void FastCopy(ReadOnlySpan<byte> source, Span<byte> destination)
    {
        if (Vector256.IsHardwareAccelerated && source.Length >= 32)
        {
            // 使用 AVX2 一次拷贝 32 字节
            var remaining = source.Length;
            var offset = 0;

            while (remaining >= 32)
            {
                var vec = Vector256.Create(source.Slice(offset, 32));
                vec.CopyTo(destination.Slice(offset, 32));
                offset += 32;
                remaining -= 32;
            }

            // 处理剩余字节
            source.Slice(offset).CopyTo(destination.Slice(offset));
        }
        else
        {
            source.CopyTo(destination);
        }
    }
}
```
