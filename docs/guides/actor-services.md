# Actor 服务开发

Actor 服务适合承载玩家、房间、战斗、公会等有身份、有状态、需要串行化处理的游戏逻辑。

## 基本步骤

1. 在共享契约项目定义 Hub 接口。
2. 在服务端实现该接口，并继承 `PulseServiceBase`。
3. 为服务选择稳定的 `ServiceName` 和 `ServiceId`。
4. 在 DI 中注册服务或服务工厂。
5. 编写单元测试覆盖生命周期、并发调用和异常路径。

## 状态设计

- 内存态适合缓存、房间运行态和临时会话。
- 关键业务状态应持久化到数据库或事件流。
- 需要跨节点迁移内存态时，实现 `IActorStateSnapshot`。
- 迁移流程必须显式触发，不要假设集群启动后自动迁移所有 Actor。

## 并发建议

- 同一 Actor 内部避免裸 `Task.Run` 修改状态。
- 外部 IO 应设置超时和取消。
- 扣费、发奖、库存等操作必须业务幂等。
- 高并发广播应优先走框架提供的扇出和路由能力。
