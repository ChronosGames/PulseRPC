# PulseRPC 协议号自动生成设计文档

## 文档信息

- **版本**: 1.0

- **作者**: PulseRPC Team
- **状态**: 设计中

## 1. 概述

本文档描述 PulseRPC 框架中协议号（ProtocolId）的自动生成机制。通过一致性哈希算法，使协议号对业务层完全透明，客户端和服务端自动对齐。

### 1.1 设计目标

- ✅ **业务层无感**：用户只定义接口，无需手动分配协议号
- ✅ **自动对齐**：客户端和服务端使用相同算法，保证协议号一致
- ✅ **类型安全**：编译时生成代码，避免运行时错误
- ✅ **版本隔离**：接口签名变化时，协议号自动变化
- ✅ **碰撞检测**：编译时检测协议号冲突

### 1.2 核心原理

```
接口方法签名 → 一致性哈希 → ProtocolId (ushort)
                    ↓
客户端和服务端使用相同算法，保证协议号一致
```

## 2. 协议号生成算法

### 2.1 核心实现

```csharp
// PulseRPC.Abstractions/ProtocolIdGenerator.cs
public static class ProtocolIdGenerator
{
    /// <summary>
    /// 根据方法全限定名生成协议号
    /// </summary>
    public static ushort Generate(string fullyQualifiedMethodName)
    {
        // 使用 xxHash32 保证一致性和性能
        var hash = XxHash32.HashToUInt32(Encoding.UTF8.GetBytes(fullyQualifiedMethodName));

        // 取低 16 位作为协议号
        return (ushort)(hash & 0xFFFF);
    }

    /// <summary>
    /// 生成方法全限定名
    /// 格式：{InterfaceFullName}.{MethodName}({ParamType1},{ParamType2},...)
    /// </summary>
    public static string GetMethodSignature(IMethodSymbol method)
    {
        var interfaceName = method.ContainingType.ToDisplayString();
        var methodName = method.Name;
        var parameters = string.Join(",",
            method.Parameters.Select(p => p.Type.ToDisplayString()));

        return $"{interfaceName}.{methodName}({parameters})";
    }
}
```

### 2.2 生成示例

| 接口方法 | 方法签名 | Hash 值 | ProtocolId |
|---------|---------|---------|-----------|
| `IChatHub.SendMessageAsync(string)` | `MyApp.IChatHub.SendMessageAsync(string)` | 0x00003A7F | 14975 |
| `IChatHub.GetHistoryAsync(int)` | `MyApp.IChatHub.GetHistoryAsync(int)` | 0x00005B79 | 23401 |
| `IPlayerHub.LoginAsync(string, string)` | `MyApp.IPlayerHub.LoginAsync(string,string)` | 0x00007D2C | 32044 |

### 2.3 算法选择

| 算法 | 性能 | 碰撞率 | 说明 |
|------|------|--------|------|
| MD5 | 中 | 极低 | 过于复杂，不适合编译时 |
| xxHash32 | 极高 | 低 | **推荐**，快速且碰撞率低 |
| MurmurHash3 | 高 | 低 | 备选方案 |
| CRC32 | 高 | 中 | 碰撞率较高 |

```csharp
                GetHistoryAsync_ProtocolId,
                payload,
                CancellationToken.None);

            var response = _serializer.Deserialize<GetHistoryAsyncResponse>(responsePayload.Span);
            return response.Result;
        }

        {
            public string Message { get; set; } = string.Empty;

        [MemoryPackable]
        internal partial class GetHistoryAsyncRequest
        {
            public int Count { get; set; }
        }

        [MemoryPackable]
        internal partial class GetHistoryAsyncResponse
        {
            public ChatMessage[] Result { get; set; } = Array.Empty<ChatMessage>();
```

```csharp
#nullable enable

namespace MyApp.Contracts.Generated
{
    using System;
    using System.Threading;
    using System.Threading.Tasks;
    using MemoryPack;
    using PulseRPC.Client;

    /// <summary>
    /// 自动生成的 IChatReceiver 分发器
    /// </summary>
    internal sealed partial class ChatReceiverDispatcher
    {
        // 协议号
        // Hash("MyApp.Contracts.IChatReceiver.OnMessageReceived(MyApp.Contracts.ChatMessage)") = 45231
        private const ushort OnMessageReceived_ProtocolId = 45231;

        // Hash("MyApp.Contracts.IChatReceiver.OnUserJoined(string)") = 32108
        private const ushort OnUserJoined_ProtocolId = 32108;

        private readonly IChatReceiver _receiver;
        private readonly IPulseRPCSerializer _serializer;
        private readonly IMessageDispatcher _dispatcher;

        public ChatReceiverDispatcher(
            IChatReceiver receiver,
            IPulseRPCSerializer serializer,
            IMessageDispatcher dispatcher)
        {
            _receiver = receiver ?? throw new ArgumentNullException(nameof(receiver));
            _serializer = serializer ?? throw new ArgumentNullException(nameof(serializer));
            _dispatcher = dispatcher ?? throw new ArgumentNullException(nameof(dispatcher));
        }

        /// <summary>
        /// 注册协议号处理器到连接
        /// </summary>
        public void Register(IPulseConnection connection)
        {
            connection.RegisterHandler(OnMessageReceived_ProtocolId, HandleOnMessageReceivedAsync);
            connection.RegisterHandler(OnUserJoined_ProtocolId, HandleOnUserJoinedAsync);
        }

        private async ValueTask HandleOnMessageReceivedAsync(ReadOnlyMemory<byte> payload)
        {
            // 1. 反序列化
            var data = _serializer.Deserialize<OnMessageReceivedData>(payload.Span);

            // 2. 调度到 Unity 主线程
            await _dispatcher.DispatchAsync(OnMessageReceived_ProtocolId, payload, default);

            // 3. 调用用户实现
            _receiver.OnMessageReceived(data.Message);
        }

        private async ValueTask HandleOnUserJoinedAsync(ReadOnlyMemory<byte> payload)
        {
            var data = _serializer.Deserialize<OnUserJoinedData>(payload.Span);
            await _dispatcher.DispatchAsync(OnUserJoined_ProtocolId, payload, default);
            _receiver.OnUserJoined(data.Username);
        }

        // 自动生成的数据包装类型
        [MemoryPackable]
        internal partial class OnMessageReceivedData
        {
            public ChatMessage Message { get; set; } = null!;
        }

        [MemoryPackable]
        internal partial class OnUserJoinedData
        {
            public string Username { get; set; } = string.Empty;
        }
    }
}
```

#### 生成的路由表

```csharp
// PulseRPC.Server.SourceGenerator 自动生成
// <auto-generated />
#nullable enable

namespace MyApp.Contracts.Generated
{
    using System;
    using System.Threading;
    using System.Threading.Tasks;
    using MemoryPack;
    using PulseRPC.Server;

    /// <summary>
    /// 自动生成的 IChatHub 路由表
    /// </summary>
    internal sealed partial class ChatHubRouter
    {
        // 协议号（与客户端一致）
        private const ushort SendMessageAsync_ProtocolId = 14975;
        private const ushort GetHistoryAsync_ProtocolId = 23401;

        private readonly IChatHub _hub;
        private readonly IPulseRPCSerializer _serializer;

        public ChatHubRouter(IChatHub hub, IPulseRPCSerializer serializer)
        {
            _hub = hub ?? throw new ArgumentNullException(nameof(hub));
            _serializer = serializer ?? throw new ArgumentNullException(nameof(serializer));
        }

        /// <summary>
        /// 注册路由到 Service
        /// </summary>
        public void Register(IHubRegistry registry)
        {
            registry.RegisterHandler(SendMessageAsync_ProtocolId, HandleSendMessageAsyncAsync);
            registry.RegisterHandler(GetHistoryAsync_ProtocolId, HandleGetHistoryAsyncAsync);
        }

        /// <summary>
        /// 处理消息（由 Service 调用）
        /// </summary>
        public async ValueTask<ReadOnlyMemory<byte>> HandleAsync(
            ushort protocolId,
            ReadOnlyMemory<byte> payload,
            IHubContext context,
            CancellationToken ct)
        {
            return protocolId switch
            {
                SendMessageAsync_ProtocolId => await HandleSendMessageAsyncAsync(payload, context, ct),
                GetHistoryAsync_ProtocolId => await HandleGetHistoryAsyncAsync(payload, context, ct),
                _ => throw new InvalidOperationException($"Unknown protocol ID: {protocolId}")
            };
        }

        private async ValueTask<ReadOnlyMemory<byte>> HandleSendMessageAsyncAsync(
            ReadOnlyMemory<byte> payload,
            IHubContext context,
            CancellationToken ct)
        {
            // 1. 反序列化请求
            var request = _serializer.Deserialize<SendMessageAsyncRequest>(payload.Span);

            // 2. 调用 Hub 方法
            await _hub.SendMessageAsync(request.Message);

            // 3. 单向消息，无需响应
            return ReadOnlyMemory<byte>.Empty;
        }

        private async ValueTask<ReadOnlyMemory<byte>> HandleGetHistoryAsyncAsync(
            ReadOnlyMemory<byte> payload,
            IHubContext context,
            CancellationToken ct)
        {
            // 1. 反序列化请求
            var request = _serializer.Deserialize<GetHistoryAsyncRequest>(payload.Span);

            // 2. 调用 Hub 方法
            var result = await _hub.GetHistoryAsync(request.Count);

            // 3. 序列化响应
            var response = new GetHistoryAsyncResponse { Result = result };
            return _serializer.Serialize(response);
        }

        // 自动生成的请求/响应类型（与客户端一致）
        [MemoryPackable]
        internal partial class SendMessageAsyncRequest
        {
            public string Message { get; set; } = string.Empty;
        }

        [MemoryPackable]
        internal partial class GetHistoryAsyncRequest
        {
            public int Count { get; set; }
        }

        [MemoryPackable]
        internal partial class GetHistoryAsyncResponse
        {
            public ChatMessage[] Result { get; set; } = Array.Empty<ChatMessage>();
        }
    }
}
```

```csharp
// 用户定义的 Service
public partial class ChatService : PulseServiceBase
{
    private readonly ChatHub _hub;
    private readonly ChatHubRouter _router;
    private readonly Dictionary<string, ChatReceiverSender> _clients = new();

    public ChatService(PID serviceId, IPulseRPCSerializer serializer)
        : base(serviceId)
    {
        _hub = new ChatHub(this);
        _router = new ChatHubRouter(_hub, serializer);
        Start();

    protected override async ValueTask<ReadOnlyMemory<byte>> OnMessageAsync(
        ushort protocolId,
        ReadOnlyMemory<byte> payload,
        IHubContext context,
        CancellationToken ct)
    {
        // 使用生成的路由表处理
        return await _router.HandleAsync(protocolId, payload, context, ct);

    {
        var chatMessage = new ChatMessage
        {
            Content = message,
            Timestamp = DateTime.UtcNow
        };

        // 使用生成的发送代理推送到所有客户端
        foreach (var sender in _clients.Values)
        {
            await sender.OnMessageReceivedAsync(chatMessage);
        }
    }
```

## 5. 协议号碰撞处理

### 5.1 编译时检测

```csharp
// PulseRPC.Client.SourceGenerator
public class ProtocolIdCollisionDetector
{
    private readonly Dictionary<ushort, string> _protocolMap = new();

    public void Register(ushort protocolId, string methodSignature, Location location, SourceProductionContext context)
    {
        if (_protocolMap.TryGetValue(protocolId, out var existingSignature))
        {
            // 生成编译错误
            var diagnostic = Diagnostic.Create(
                new DiagnosticDescriptor(
                    id: "PRPC001",
                    title: "Protocol ID collision detected",
                    messageFormat: "Protocol ID {0} collision: '{1}' conflicts with '{2}'. " +
                                   "Consider using [Protocol(ushort)] attribute to manually assign a unique ID.",
                    category: "PulseRPC",
                    defaultSeverity: DiagnosticSeverity.Error,
                    isEnabledByDefault: true
                ),
                location,
                protocolId,
                methodSignature,
                existingSignature
            );

            context.ReportDiagnostic(diagnostic);
        }
        else
        {
            _protocolMap[protocolId] = methodSignature;
        }
    }
}
```

### 5.2 碰撞示例

```csharp
// 假设这两个方法产生了相同的哈希值（极低概率）
public interface IHub1 : IPulseHub
{
    ValueTask MethodA(); // Hash = 12345

public interface IHub2 : IPulseHub

```

## 6. 完整调用流程

### 6.1 客户端调用 Hub 方法

```
1. 用户代码
   ↓
   await chatHub.SendMessageAsync("Hello");

2. 生成的代理
   ↓
   var request = new SendMessageAsyncRequest { Message = "Hello" };
   var payload = serializer.Serialize(request);
   await connection.SendAsync(14975, payload); // protocolId: 14975

3. 传输层
   ↓
   TCP/KCP Transport 发送 [14975][payload]

4. 服务端接收
   ↓
   ServiceIsolatedDispatcher 解析 protocolId: 14975

5. 路由到 Service
   ↓
   ChatService.EnqueueAsync(14975, payload, context)

6. Service 消息队列处理
   ↓
   ChatHubRouter.HandleAsync(14975, payload, context)

7. 反序列化并调用
   ↓
   var request = deserialize(payload);
   await ChatHub.SendMessageAsync(request.Message);
```

### 6.2 服务端推送 Receiver 事件

```
1. 服务端业务逻辑
   ↓
   await sender.OnMessageReceivedAsync(message);

2. 生成的发送代理
   ↓
   var data = new OnMessageReceivedData { Message = message };
   var payload = serializer.Serialize(data);
   await connection.SendAsync(45231, payload); // protocolId: 45231

3. 传输层
   ↓
   TCP/KCP Transport 发送 [45231][payload]

4. 客户端接收
   ↓
   Connection 根据 protocolId 查找处理器

5. 调用注册的处理器
   ↓
   ChatReceiverDispatcher.HandleOnMessageReceivedAsync(payload)

6. 反序列化
   ↓
   var data = deserialize(payload);

7. 调度到 Unity 主线程
   ↓
   UnityMainThreadDispatcher.DispatchAsync()

8. 调用用户实现
   ↓
   MyChatReceiver.OnMessageReceived(data.Message);
```

## 7. 接口规范

### 7.1 IPulseHub 定义规范

```csharp
// ✅ 正确：异步方法
public interface IChatHub : IPulseHub
{
    ValueTask SendMessageAsync(string message);              // 单向消息
    ValueTask<ChatMessage[]> GetHistoryAsync(int count);     // 请求-响应
}

// ❌ 错误：同步方法
public interface IBadHub : IPulseHub
{
    void SendMessage(string message);        // 不支持同步方法
    ChatMessage[] GetHistory(int count);     // 不支持同步方法
}

// ❌ 错误：方法重载
public interface IBadHub2 : IPulseHub
{
    ValueTask SendAsync(string message);
    ValueTask SendAsync(int code);           // 不支持重载（协议号会冲突）
}
```

### 7.2 IPulseReceiver 定义规范

```csharp
// ✅ 正确：同步方法（在 Unity 主线程执行）
public interface IChatReceiver : IPulseReceiver
```

## 12. 最佳实践

### 12.1 接口设计原则

✅ **推荐做法**
- 使用清晰的接口命名（`IChatHub`, `IPlayerHub`）
- 方法名使用 `Async` 后缀
- 参数类型使用 `[MemoryPackable]` 标记
- 使用 `ValueTask` 而非 `Task`

❌ **避免做法**
- 方法重载（会导致协议号冲突）
- 使用 `object` 或 `dynamic` 参数
- 频繁修改接口签名（破坏兼容性）

### 12.2 版本管理

- 新增方法：兼容（添加新协议号）
- 修改参数：不兼容（使用新接口版本）
- 删除方法：不兼容（保留废弃标记）

```csharp
public interface IChatHub : IPulseHub
{
    [Obsolete("Use SendRichMessageAsync instead")]
    ValueTask SendMessageAsync(string message);

    ValueTask SendRichMessageAsync(RichMessage message);
}
```

### 12.3 性能考虑

- 使用 `ValueTask` 避免堆分配
- 使用 `ReadOnlyMemory<byte>` 避免拷贝
- 参数尽量使用值类型或小对象
- 避免传递大数组（使用流式传输）
