# 文档风格

## 分类

- Tutorial：让用户完成第一次成功体验。
- How-to：解决一个具体任务。
- Concept：解释模型、边界和权衡。
- Reference：列事实，不讲故事。
- Archive：历史背景，不作为当前 API 依据。

## 写作规则

- 当前事实必须来自源码、测试或可运行样例。
- 历史设计必须放在 `docs/archive/` 并保留状态提示。
- 不把“计划实现”写成“已经支持”。
- 代码块如果不可编译，必须标注为伪代码。
- 每篇文档只承担一种主要职责。
- 链接使用相对路径。

## 命名规则

- 新目录使用英文小写短名，例如 `concepts/`、`guides/`。
- 当前文档文件名使用英文 kebab-case。
- 归档文件名保留可理解的英文摘要，不再增加中文长文件名。
- AI Agent 文档只能放在 `.agent/`。

## 删除规则

可删除：

- 重复且已被当前文档覆盖的阶段总结。
- 未被引用、且不含当前事实的临时记录。
- 构建产物、日志、IDE/Unity 缓存。

不可直接删除：

- 当前 README 或样例入口。
- 仍被当前文档引用的说明。
- 公共 API 迁移说明。
- 兼容性策略相关记录。
